# OpenAPI ↔ Markdown Consistency (RAE)

## Endpoints

All three RAE endpoints exist in both artifacts:
- POST /{version}/milestones/{milestone_id}/rae
- GET /{version}/milestones/{milestone_id}/rae/history
- GET /{version}/projects/{project_id}/rae/summary

## Parameters

### `sort_order` default
- OpenAPI (openapi/components/parameters.yaml): `sort_order` default is `desc`.
- Markdown: aligned to default `desc`.

### `date_from` / `date_to` formats
- OpenAPI uses `date-time`.
- Markdown uses “datetime” phrasing; this is acceptable but should explicitly say ISO 8601 `date-time` for consistency.

## Schemas

### RAECreate.details.task_estimates
- OpenAPI requires `remaining_cost`.
- Markdown matches `remaining_cost`.

### RAE terminology
- OpenAPI description uses “Remaining At End” in one place.
- Markdown uses “Remaining to be Committed / Reste À Engager”.
- Recommendation: unify terminology to avoid confusion.

## Status Codes / Error handling

RAE endpoint error coverage is aligned between Markdown and OpenAPI (including 401/403/404/429/500; and 400 for POST).
